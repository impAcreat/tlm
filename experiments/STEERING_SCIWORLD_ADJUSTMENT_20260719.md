# ScienceWorld steering adjustment — 2026-07-19

## Question

Why did the initial SkillOpt text improve ScienceWorld soft score while its
prompt-conditioned steering vector did not, and can a benchmark-aware
intervention recover a held-out gain?

## Protocol and the pilot confound

- All 30 `boil` variations, `easy` simplification, Qwen3.5-4B greedy decoding,
  and 20 environment steps.
- Disjoint split: train 0--5, validation 6--11, test 12--29.
- The pilot used train/selection/test sizes 6/3/3 and extracted from the first
  12 states. Its test soft scores were bad 0.0067 and good text 0.1233.
- Powered evaluation uses six validation and 18 test variations. Vector
  extraction uses 180 states from 18 train-only trajectories generated by
  bad, good, and parser-only skills.

The first powered rerun exposed a ScienceWorld adapter confound. The Java
environment returns the same valid-action set in unstable object order, while
the runner inserted the first 180 actions verbatim into the prompt. A diff of
the same task/state across fresh environments showed identical task, history,
and observation but a different ordering of the action block; greedy actions
then diverged and cascaded. The fix preserves ScienceWorld's stable action
template/verb order and sorts only within each verb group. This preserves the
coarse benchmark prompt distribution while removing object-order noise.

Cross-GPU verification after the fix: good-text validation trajectories match
exactly on all 6/6 variations; bad-text scores match 6/6 and action sequences
match 5/6. Thus the original 0.12 textual estimate was inflated by action-order
noise. Under the stable 18-task protocol, bad/good text soft is
0.0089/0.0428 (+0.0339). On train it is 0.0150/0.0617 (+0.0467), so a smaller
but real textual signal remains.

## Benchmark-aware diagnosis

ScienceWorld emits one short `<action>` per step, unlike the longer ALFWorld
responses on which gen-only multi-layer steering was developed. Adding the same
instruction delta at three layers on every decode token produced a persistent,
state-insensitive action bias and severe repeated `look` behavior.

The stable 180-state direction remains coherent but becomes more state-specific
in later layers: cross-state cosine is 0.813 at L8, 0.707 at L14, 0.651 at L18,
and 0.613 at L22. State-conditioned kNN retrieval, early-step extraction, phase
gating, full-prefill injection, and three-layer low-dose injection were tested
on validation; none beat the simpler single-layer mid-depth intervention.

## Stable validation selection

Bad/good text soft is 0.0133/0.0200. Selected steering results from the final
180-state artifact:

| Method | Soft | Invalid | Repeat |
|---|---:|---:|---:|
| L10 gen-only, alpha 1 | 0.0133 | 0.0167 | 0.6667 |
| L14 gen-only, alpha 1 | 0.0167 | 0.0167 | 0.7281 |
| L18 gen-only, alpha 1 | 0.0133 | 0.0083 | 0.7018 |
| {14,18,22} gen-only, alpha 0.25 | 0.0133 | 0.0083 | 0.6930 |
| **L14 prefill-last + gen, alpha 1** | **0.0167** | **0.0167** | **0.6140** |

Both L14 variants reproduce the good text's `boil-8` score movement from 3 to
5. The hybrid is frozen because it has the same soft score and lower repetition.

## Frozen formal result

The frozen full-delta arm did not transfer to the 18 held-out variations:

| Arm | Soft | Delta vs bad | Improved / regressed / tied |
|---|---:|---:|---:|
| Initial text (bad) | 0.0089 | -- | -- |
| Full SkillOpt text (good) | 0.0428 | +0.0339 | 1 / 4 / 13 |
| L14 hybrid steering | 0.0089 | +0.0000 | 0 / 0 / 18 |
| Matched random | 0.0100 | +0.0011 | 1 / 0 / 17 |

Steering changed 10/18 trajectories and 2/18 first decisions, but every score
was identical to the bad-text baseline. This separates behavioral
controllability from useful task transfer.

## Procedure-component isolation

To avoid asking one vector to represent both output parsing and the boiling
procedure, a second direction was extracted as full SkillOpt text minus a
parser-only skill. It uses the same 180 train states and is distinct from the
full direction (L14 cosine 0.671). On validation, parser-only/full text were
both 0.0200 while L14 hybrid component steering reached 0.0233, combining the
best `boil-7` and `boil-8` subgoal scores. Four additional matched-random seeds
scored 0.0083, 0.0083, 0.0117, and 0.0233, so validation was promising but not
direction-specific by itself.

The frozen 18-task result was parser-only 0.0011, full text 0.0428, component
steering 0.0022, and matched random 0.0011. Additional random seeds scored
0.0011, 0.0022, 0.0011, and 0.0022: the semantic direction's one 0.02 subgoal
gain is tied by random controls and does not repair the decisive `boil-13`
trajectory. It is therefore not evidence of a steering gain.

An exact online diagnostic computed the good-minus-parser residual for the
current state rather than using a static mean. L14 prefill-only reached 0.0167
and the other tested placements were worse than parser-only 0.0200. Thus the
failure is not explained only by averaging unrelated states.

Finally, shallow multi-layer component injection tested `{8,10,12,14}` and
`{10,12,14,16}`. The best hybrid arm tied single-layer L14 at 0.0233;
generation-only was 0.0167 and both prefill-only arms were 0.0083. With no new
validation gain, no further test-set run or alpha sweep was performed.

## Conclusion

The adjusted implementation establishes a reproducible negative result:
prompt-conditioned residual directions are coherent and causally alter
ScienceWorld trajectories, but static additive last-token steering does not
recover the held-out procedural benefit of the text. Even exact per-state and
shallow multi-layer variants fail. The remaining technical hypothesis is that
the useful text effect is distributed through prompt-token attention/KV state
and multiple layers, rather than encoded as one additively reusable residual
direction. A next iteration should test KV/attention patching or a learned
distributed adapter, not a larger static-vector alpha sweep.

## Evidence boundary

- The formal unit is 18 distinct held-out variations, not repeated copies or a
  three-example smoke. All arms are run in one model process/GPU and paired by
  variation; the fourth arm is a layer/norm-matched random direction.
- This is one task family, model, learned text, and step budget. Soft gain means
  transfer of useful subgoal behavior, not full completion or cross-family gain.
- Rejected validation variants, aborted unstable-order runs, and the initial
  three-example pilot are diagnostic evidence only, not final results.
